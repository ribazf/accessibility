How We Fix the Issues

Brief fix and code snapshot for each of the six accessibility issues.

← Issues | Fixed →

1. Low color contrast

Use text and background that meet at least 4.5:1 (WCAG 2.2 AA). Check contrast (WebAIM Contrast Checker). 

.good-contrast {
  color: #1a1a1a;
  background-color: #ffffff;
  /* ~15.8:1 – passes AA */

2. Missing image description

Add meaningful alt so screen readers and failed loads have a description. 

<img src="banner.jpg" alt="Blue banner with placeholder text for accessibility example">

3. Keyboard not supported

Use a real <button> so it’s focusable and works with Enter/Space. 

<button type="button" onclick="...">Submit action</button>

4. No visible focus

Don’t remove outline; add a clear focus style so keyboard users see focus. 

a:focus-visible, button:focus-visible {
  outline: 2px solid #0066cc;
  outline-offset: 2px;
}

5. Forms without labels or clear errors

Use <label for="id">, specific error text, and aria-describedby for the error. 

<label for="email">Email address</label>
<input type="email" id="email" aria-describedby="err" aria-invalid="false" required>
<span id="err" role="alert" aria-live="polite">Please enter a valid email.</span>

6. Missing ARIA labels (icon-only button)

Give an accessible name: aria-label 

<button type="button" aria-label="Search">🔍</button>
