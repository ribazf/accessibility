1. Low color contrast

Below, text uses light gray on white. This fails WCAG 2.2 AA (requires at least 4.5:1 for normal text). Many users with low vision or in bright light cannot read it.

This paragraph has very low contrast. It is hard to read for people with visual impairments or in bright environments. Do not use light gray (#b0b0b0) on white.

2. Missing image description (no alt text)

The image below has no alt attribute. Screen readers cannot describe the image, and if the image fails to load, there is no fallback text.

3. Keyboard not supported

The “button” below is a <div> with onclick. It is not focusable with Tab and does not activate with Enter or Space. Keyboard-only and screen reader users cannot use it.

Submit action

4. No visible focus state

Links and buttons on this page have outline: none. When you tab through the page, there is no visible focus ring, so keyboard users cannot tell which element is focused.

Sample link

5. Forms without labels or clear error messages

Inputs use only placeholders (no <label>). The error message is vague (“Error”) and not associated with the field via aria-describedby. Screen reader users do not get clear guidance.

Error

6. Missing ARIA labels (icon-only button)

The search icon below has no aria-label. Screen readers only announce “button” with no context. Users cannot know it is a search button.
