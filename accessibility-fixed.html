Accessibility Demo – Fixed Version

This page corrects the same components. It should pass Lighthouse/BrowserStack accessibility checks and avoid major WCAG 2.2 AA violations.

Issues How to fix Fixed

1. Fixed: Sufficient color contrast

Text now uses dark gray (#1a1a1a) on white, which exceeds WCAG 2.2 AA (4.5:1). Comment in code notes the contrast ratio.

This paragraph has sufficient contrast. It is readable for people with low vision and in various lighting. We use dark text on a light background to meet WCAG 2.2 AA.

2. Fixed: Image with description

The image has meaningful alt text so screen readers can describe it and users get context if the image does not load.

3. Fixed: Keyboard support

A real <button> is used. It is focusable with Tab and activates with Enter and Space. No custom keyboard handling needed.

4. Fixed: Visible focus state

Focus outline is not removed. We use :focus-visible (and :focus fallback) so keyboard users see a clear focus ring.

Sample link

5. Fixed: Labels and clear error messages

Each input has a <label for="...">. Errors are specific and linked with aria-describedby. required is used where appropriate.

6. Fixed: ARIA label for icon-only button

The search button has aria-label="Search". Screen readers announce “Search, button” so users understand the control without visible text.

7. Fixed: In tab order with tabindex

Add tabindex="0" so the element is focusable with Tab. Add a keydown handler so Enter/Space activate it like a button.

8. Fixed: Role for heading

Add role="heading" and aria-level (1–6) so screen readers announce it as a heading and include it in the document outline. Prefer using a real <h1>–<h6> when possible.
